# Soapbox Event Timer

イベント進行用のローカルダッシュボードです。  
ローカル LAN 上で動かし、ダッシュボード表示と設定画面から進行管理を行う想定です。

## 概要

- ダッシュボード画面
  - 現在時刻
  - 全体の進行差分（押し / 巻き）
  - 全体消化率
  - 最終イベント終了までの残り時間
  - 現在進行イベントの表示
  - 俯瞰スケジュール表示
  - 独立タイマー 2 本

- 設定画面
  - スケジュール編集
  - 独立タイマー初期値の編集
  - ダッシュボード見た目設定
  - イベント種別ごとのカラー設定
  - 各イベントの強制同期ボタン表示 ON/OFF
  - JSON スナップショット確認

## 技術構成

- Node.js
- Express
- Socket.IO
- フロントエンドはプレーンな HTML / CSS / JavaScript

## 起動方法

依存関係が入っている前提で、以下を実行します。

```bash
npm start
```

開発時も同じです。

```bash
npm run dev
```

起動後、以下にアクセスします。

- ダッシュボード: `http://localhost:3333/`
- 設定画面: `http://localhost:3333/schedule.html`

## データファイル

データは `data/` 配下の JSON に保存されます。

- `data/state.json`
  - グローバルオフセット
  - 独立タイマー状態
  - ダッシュボード設定

- `data/schedule.json`
  - イベントスケジュール本体

## イベント種別

現在対応している種別:

- `normal`
- `break`
- `buffer`
- `special`
- `race`

## 主な操作

- オフセットを増減して進行差分を調整
- 現在イベントをプレビューで前後移動
- 強制的に前後イベントへ同期
- 独立タイマーの Start / Pause / Reset
- 設定画面からスケジュールとダッシュボード設定を保存

## API の例

- `GET /api/bootstrap`
  - 状態とスケジュールの初期取得

- `PUT /api/schedule`
  - スケジュール保存

- `PUT /api/timers`
  - 独立タイマー初期値保存

- `PUT /api/dashboard-config`
  - ダッシュボード表示設定保存

- `POST /api/offset`
  - グローバルオフセット変更

- `POST /api/resync`
  - 指定イベントへ強制同期

- `POST /api/schedule/shift`
  - 前後イベントへ強制移動

## メモ

- データを初期化したい場合は設定画面の「モックデータを初期化」を使ってください。
- 現在は簡易モック実装寄りなので、認証や権限制御は入っていません。
